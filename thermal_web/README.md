# 热成像网页查看器

STM32N6 热成像串口网页上位机。

## 启动

```powershell
python thermal_web/app.py
```

浏览器打开：

```text
http://127.0.0.1:8000
```

## 当前默认值

- 串口协议：`TEMP14`
- 默认波特率：`2000000`
- 当前实时热像数据刷新率：约 `5 fps`
- 图库能力：支持 `同步图库`、`下载最新`、`下载选中`
- 大图下载：前端最长等待 `120 秒`，页面会持续显示已等待时间

## 使用方法

1. 先确认板子已通过 `USB OTG + CH340` 连到电脑。
2. 打开网页后，点击 `刷新`。
3. 串口列表里选中对应设备，点击 `连接`。
4. 页面会实时显示热图、中心温度、最高温和最低温。
5. 需要时可切换伪彩、调温标、暂停或截图。
6. 点击 `热场分析` 会冻结当前显示帧并打开 3D 曲面；拖动旋转、滚轮缩放、双击复位视角，曲面颜色复用当前伪彩与温标。

## 说明

- 默认使用 `TEMP14` 协议，和 `tools/uart_temp14_parser.py` 保持一致；实时预览包头当前上报 `128x96`。
- 当前默认波特率已经是 `2000000`，与当前固件默认值一致。
- 当前网页端看到的实时热像数据更新速率，主要由固件串口温度流发送周期决定；当前默认约为 `5 fps`，实时流尺寸为 `128x96`。
- 3D 热场在浏览器端从当前 `temp14` 帧生成，不新增固件或串口负担；Z 轴随温度单调升高，交互曲面采用降采样网格以保证拖动流畅。
- 串口文件接收采用分块缓冲读取，并按 `FILE_BEGIN` 声明的总字节数收包，不再依赖固定 DATA 分块大小；可同时兼容旧固件的 `384 byte` 分块和新固件的 `1536 byte` 分块。
- 下载 640×480 截图时请保持串口连接；完成后页面会显示实际耗时和本地下载链接，不会再额外触发一次图库同步。
- 页面支持自动选择首个可用串口。
- 如果串口被别的软件占用，网页无法连接。
